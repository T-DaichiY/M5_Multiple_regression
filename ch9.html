Chapter 9

➡️ 変数選択：前進選択（Forward）

20個の予測変数があると、組み合わせは100万通り以上。全部試すのは無理。「何もない状態」から最良の変数を1つずつ足していく、賢い選び方を学ぶ。

① 変数選択とは ② なぜ全探索は無理か ③ 前進選択の手順 ④ step関数で自動化 ⑤ BICに切り替える 📝 Quiz
🍎 むずかしいと感じたら？ 図・アニメ・スライダーで“特性だけ”をつかめる ▶ 第9章 超やさしい版 を先に読むのがおすすめです。
①

変数選択とは

データに埋もれた「シグナル」を見つける

モデルを作るとき、私たちはデータに隠れたパターン（シグナル）を見つけたい。変数選択は、モデルにとって最良の予測変数＝データを最もよく説明し、予測に役立つ変数を選ぶプロセスです。

ただし「最良」の定義は、組織がどう「良さ」を定義し、どの指標を使うかによります。

🍎 かんたん説明

探偵が事件を解くとき、手がかりは多ければいいわけじゃない。本当に役立つ手がかりだけを選びますよね。変数選択は「使える手がかりだけを厳選する」探偵の技です🔍

②

なぜ「全部試す」は無理なのか

全部分集合回帰の組み合わせ爆発

理屈の上では、すべての変数の組み合わせを試す全部分集合回帰（all subset regression）が一番正確です。でも計算量が爆発的に増えます。

予測変数の数試すべきモデルの数（2ⁿ）
3個8通り（手作業でも可能）
10個2¹⁰ ＝ 1,024通り
20個100万通り超！

20個でも100万通り超。これは現実的ではありません。そこで、もっと効率的な前進選択（forward selection）を使います。

🍎 かんたん説明

20種類のトッピングから「最高の組み合わせ」を全部試食したら、100万通り…お腹がパンクします🍕 だから「一番おいしいトッピングを1つずつ足していく」という賢い方法を使うのです。

③

前進選択の手順

何もない状態から1つずつ足していく
ステップ1：各変数を単独で試す → 最良の1つを選ぶ
↓
ステップ2：その変数＋2つ目の候補を試す → 最良の2つ目を追加
↓
これを繰り返す
↺ 停止条件（指標が改善しなくなる）まで繰り返す

手動の例：湿度・気温・風速・「夏かどうか」を単独で試すと、気温がR²で最も良い（ドメイン知識的にも納得）。だからステップ1で気温を選択。

# ステップ1: 単独で一番良いのは気温（R²最大）lm(rentals ~ temperature, data = bikes) # R²が一番高い → 採用# ステップ2: 気温＋2つ目を試す（調整済みR²で比較）lm(rentals ~ temperature + humidity, data = bikes)# 0.44 → 0.488 に改善 → 湿度も追加
📌 R² か 調整済みR² か：ステップ1（予測変数1つ）はR²。ステップ2以降（2つ以上）は調整済みR²を使います（複数予測変数のときは調整済みが正しい）。
🍎 かんたん説明

前進選択は「空っぽのリュックに、一番役立つ道具から順に詰めていく」やり方。まず一番大事な道具（気温）、次に2番目（湿度）…。もう足しても便利にならなくなったら、そこでストップ🎒

④

step関数で自動化

AICを最小化しながら自動で変数を追加

手作業は大変なので step() 関数で自動化します。step関数はデフォルトでAICを最小化します（R²ではなく）。AICは当てはまりと複雑さのバランスを取り、シンプルなモデルを好みます。

# ① 空のモデル（切片だけ = 予測変数なし）null_model <- lm(rentals ~ 1, data = bikes)# ② 全部入りモデル（~ . で全変数）full_model <- lm(rentals ~ ., data = bikes)# ③ 前進選択: nullから始めてfullまで足していくstep(null_model, scope = list(lower = null_model, upper = full_model), direction = "forward")
⚠️ AICの読み方：AICは相対的に使います。最初のnullモデルのAIC（例：10,000）の絶対値自体に意味はありません。step関数は、各変数を試してAICが一番下がるものを順に追加し、AICが増え始めたら（=もう改善しない）停止します。例では最終的にAIC 9367で停止しました。

処理の流れ：null → 一番AICを下げる変数（real_feel）を追加 → 次にdate → 次にhumidity²…と続き、これ以上どの変数を足してもAICが増えてしまう点で停止。それが前進選択の最終モデルです。

🍎 かんたん説明

step関数は「自動で道具を詰めてくれるロボット」。役立つ道具（AICを下げる）を順に詰め、「もう何を足してもリュックが重くなるだけ」になったら自動で手を止めてくれます🤖

⑤

停止基準をBICに切り替える

もっとシンプルなモデルが欲しいとき

AICの代わりにBICを使うこともできます。BICは複雑さへのペナルティがより強く、よりシンプルなモデルを選びます。Rでは k = log(行数) を指定します。

# BICで前進選択（k = log(行数)）step(null_model, scope = list(lower = null_model, upper = full_model), direction = "forward", k = log(nrow(bikes))) # これでBICになる
📌 注意：出力には "AIC" と表示されますが、k=log(n) を指定したので実際はBICです。BICのほうが少ない予測変数を選ぶ傾向があります。
⚠️ 前進選択の弱点：前進選択（と次章の後退選択）は多重共線性を考慮しません。すでに似た変数が入っていても、相関の高い変数を追加してしまい、冗長になることがあります。だからモデルを作った後にVIF（第4章）で共線性をチェックし、より簡潔（パーシモニアス）なモデルにするとよいでしょう。
🍎 かんたん説明

AICは「ふつうの先生」、BICは「もっと厳しくシンプルを求める先生」。同じ前進選択でも、BIC先生に頼むと「最小限の道具だけ」を選んでくれます。ただしどちらも「似た道具のダブり（共線性）」は見てくれないので、最後は自分でVIFチェックを！👩‍🏫
