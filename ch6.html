🏷️ カテゴリ変数の変換

「季節＝1」「曜日＝2」…数字で表されたカテゴリ。そのまま使うと意味がわからないし、特定の季節だけ取り除くこともできない。factor化とワンホットエンコーディングで解決する。

①

なぜカテゴリ変数を変換するのか

2つの理由：①わかりやすさ ②特定カテゴリだけ操作したい

これまで気温・風速など量的な予測変数だけを使ってきました。この章ではカテゴリ予測変数（季節・祝日・曜日・天気など）を扱います。まず、文字データは（自由記述テキストでなければ）factor型に変換するのが基本です。

変換する2つの理由

理由説明
① わかりやすさ「季節＝1、曜日＝6、天気＝2」では意味不明。コードブック（データ辞書）を見ないとわからない。グラフでも曜日が数字だと読み手に伝わらない。
② 特定カテゴリの操作「天気1は残して天気3だけ削りたい」ができない。数字のままだと1つのカテゴリだけ取り除けない。
📌 余談：カテゴリ変数（季節・祝日・曜日・天気）をモデルに加えると、調整済みR²が前のモデル（約0.6）から増加しました。調整済みR²は無駄な変数を罰するので、増えたということは、これらが役立つ情報を持っている証拠です。
🍎 かんたん説明

「給食メニューが1番、2番、3番」と数字で言われても何が出るかわからないですよね。「1＝カレー、2＝ラーメン」と名前にすればすぐわかる。カテゴリ変数の変換は、数字に「名前」をつけてあげる作業です🍛

②

factor化と case_when で名前をつける

数字を意味のあるラベルに置き換える

case_when を使って、数字を意味のある名前に変換します。== は2つ（イコール2個）、スペースに注意、最後に .default で取りこぼしを防ぎます。

bikes <- bikes %>% mutate(season = case_when( season == 1 ~ "winter", season == 2 ~ "spring", season == 3 ~ "summer", season == 4 ~ "fall", .default = as.character(season)))

同じように、祝日（0＝いいえ, 1＝はい）、曜日（0＝日曜〜6＝土曜）、天気（1＝晴れ, 2＝降水…）も変換できます。すると最初の行が「冬・祝日でない・土曜・小雨」のように一目で読めるようになります。

📌 補足：変換後の文字列は、改めてfactor型に戻しておくのがおすすめです（順序づけや回帰での扱いがスムーズになります）。
🍎 かんたん説明

case_whenは「翻訳機」。「1という暗号は冬、2は春…」というルールを教えると、数字の羅列を人間が読める言葉に翻訳してくれます🌸

③

ワンホット（ダミー）エンコーディング

1つの列を「はい/いいえ」の複数の列に分ける

特定のカテゴリ（例：夏だけ）をモデルから削るには、ワンホットエンコーディング（ダミーエンコーディング）が必要です。1つの「季節」列を、季節ごとの「はい/いいえ」列に分けます。

ワンホットエンコーディングのしくみ
1つの「season」列が、季節ごとの「はい(1)/いいえ(0)」の列に分かれる。これで「夏だけ削る」が可能に。
# if_else で手動ワンホット（例：冬かどうか）bikes <- bikes %>% mutate(season_winter = if_else(season == "winter", 1, 0), season_spring = if_else(season == "spring", 1, 0), season_summer = if_else(season == "summer", 1, 0))

これで「夏」だけを回帰から取り除いて、冬・春・秋は残す、といった操作ができます。fastDummies パッケージでも自動でできますが、課題ではこの手動アプローチを使うこと（fastDummiesは不可）。

🍎 かんたん説明

1個のスイッチ（season）を、「冬スイッチ」「春スイッチ」「夏スイッチ」「秋スイッチ」の4個に分けるイメージ。今日が夏なら「夏スイッチだけON(1)、他はOFF(0)」。こうすれば「夏スイッチだけ外す」ことができます🔘

④

ダミー変数トラップ（n−1の法則）

最初の1列を必ず取り除く

大事なルール：カテゴリがn個なら、ダミー列はn個ではなく n−1個にします。1つは必ず取り除く（remove_first_dummy）。

⚠️ なぜ1つ削る？ 季節4つで「春でも夏でも冬でもない」なら、自動的に秋に決まります。だから「秋」列は無くても情報が完全。むしろ残すと多重共線性が起きてしまいます。これをダミー変数トラップと呼びます。
カテゴリカテゴリ数必要なダミー列
季節4（春夏秋冬）3（1つ削除）
曜日76（1つ削除）
天気3（晴れ/小雨/大雨）2（1つ削除）
📌 トラップに落ちたサイン：係数に NA が出たり、"singularity" というメッセージが出たら、ダミー変数トラップに落ちています。n−1個にすれば解決します。
🍎 かんたん説明

「右でも左でも真ん中でもないなら…必ず上ですよね」。だから「上」のスイッチは省略できる。全部のスイッチを用意すると逆に矛盾（共線性）が起きるので、1つ減らすのがルールです🔼

⑤

数値とカテゴリが混在するときのスケーリング

比較しやすくするための仕上げ

数値変数（気温・湿度）とワンホット変数（曜日など）が混在する場合、比較しやすくするため数値変数のスケーリングを検討します。

  • 標準化（Standardization）：Zスコアを使う（第3章で学んだ方法）。
  • 正規化（Normalization）：最大・最小でスケールする方法。

これらは機械学習の授業でより詳しく学びます。機械学習モデルでは、スケーリングが必要になることが多いです。

🍎 かんたん説明

身長(cm)と「夏スイッチ(0か1)」を一緒に扱うと、桁が違いすぎて比べにくい。みんなを「0〜1」や「偏差値」にそろえると、フェアに比べられます。これがスケーリングです⚖️

📘 重要用語

カテゴリ変数
Categorical Variable
季節・曜日・天気など、種類を表す変数。数字でも実体はカテゴリ。
factor型
factor
Rでカテゴリを扱う型。回帰や可視化で適切に処理される。
ワンホットエンコーディング
One-Hot / Dummy Encoding
1列を「はい/いいえ(1/0)」の複数列に分割すること。
ダミー変数トラップ
Dummy Variable Trap
全カテゴリをダミー化して共線性が起きる罠。n−1列にして回避。
case_when
case_when()
条件に応じて値を置き換えるdplyr関数。数字→ラベル変換に使う。
