モデルは作れた。でも「見たことのない新しいデータでもうまくいく？」が本当の問い。訓練・テスト分割で「汎化（はんか）」を確かめ、過学習を見抜く方法を学ぶ。最終章！

①

なぜ「性能の評価」が必要なのか

本当に知りたいのは「未知のデータでの成績」

これまで診断（第2章）でモデルの選び方を学びました。でも本当に知りたいのは——このモデル、見たことのない新しいデータでもうまく予測できる？ ということです。

モデルを実際に本番（production）で使うなら、明日来る新しいデータ、今はまだ存在しないデータに対しても、ちゃんと予測してほしい。「学習に使ったデータ」だけでなく「まったく新しいデータ」で良い成績を出せるか——これが汎化（generalization）です。

📌 汎化（はんか）とは：モデルが未知のデータにもうまく対応できる能力のこと。本番で使うモデルは、この汎化性能が高くなければ意味がありません。
🍎 かんたん説明

テスト勉強で「過去問の答えを丸暗記」した人は、過去問では満点でも、本番の新しい問題では解けませんよね。モデルも同じ。「練習データで満点」より「本番の新データで何点取れるか」が大事なのです📝

②

過学習（オーバーフィッティング）

「丸暗記」して「理解」していない状態

データサイエンティストが向き合う大問題が過学習（overfitting）です。これは、モデルがパターン（シグナル）を理解するのではなく、手元のデータを丸暗記してしまう状態です。

過学習したモデルは、ノイズ（雑音・たまたまのブレ）をシグナル（本当のパターン）だと勘違いします。すると、練習データには完璧に合うのに、新しいデータではボロボロになります。

過学習：練習では満点、本番でガタ落ち
📌 過学習の見分け方：訓練データでR²＝0.95（とても高い）なのに、テストデータでR²＝0.4（低い）。この大きなギャップが過学習のサインです。逆に、訓練とテストのR²がだいたい同じなら、過学習していない良いモデルです。
⚠️ 補足：単純な重回帰では過学習はそこまで大きな問題にならないことが多いです。でも今後学ぶ機械学習モデルはもっと複雑で、丸暗記しやすい（過学習しやすい）ので、この考え方がとても重要になります。
🍎 かんたん説明

過学習は「教科書を丸暗記したけど、意味は理解していない」状態。同じ問題なら満点だけど、ちょっとひねられると解けない。モデルにも「丸暗記タイプ」と「理解タイプ」がいて、私たちは理解タイプを作りたいのです🧠

③

訓練・テスト分割（Train/Test Split）

データを2つに分けて、片方は「本番テスト」用にとっておく

過学習を見抜く道具が訓練・テスト分割です。データを2つに分けます。

全データ
731行
→
訓練セット（80%）
モデルを学習させる
テストセット（20%）
性能を評価する
  • 訓練セット（training set）：モデルにパターンを学ばせるためのデータ。
  • テストセット（test set）：取っておいて、モデルが見たことのないデータとして性能を評価する。「本番」の代わり。

このクラスでは80:20分割（80%を訓練、20%をテスト）を使います。他のコースでは70:30、85:15などもあります。データが少ないときは分割すべきではありません（学習に使えるデータが減りすぎるため）。

📌 評価セット・ホールドアウトセット：もっと高度には「評価（validation）セット」や「ホールドアウトセット」もありますが、この章では入門として訓練・テストの2分割に絞ります。
🍎 かんたん説明

試験勉強で問題集を「練習用」と「模擬試験用」に分けるのと同じ。練習用で勉強して、模擬試験用（取っておいた問題）で実力を測る。最初から答えを見ちゃダメ＝テストセットは最後まで取っておきます📚

④

Rで実装する

seedを固定して、ランダムに80:20に分ける

分割にはランダム性があるので、再現性のために set.seed() でシードを固定します。これでチームの誰が実行しても同じ結果になります。

# ① 再現性のためにシードを固定set.seed(123)# ② 行数を取得（bikesの行数 = 731）n <- dim(bikes)[1]# ③ 80%の行をランダムに選び、その番号を保存train_indices <- sample(1:n, size = 0.8 * n)# ④ 訓練セット＝選ばれた行、テストセット＝それ以外train_set <- bikes[train_indices, ]test_set <- bikes[-train_indices, ]

結果、訓練セットは約584行（80%）、テストセットは約147行（20%）になります（ちょうど80:20にはならないこともあります）。

訓練 → 予測 → 評価の流れ

# ① 訓練セットでモデルを作るmodel_train <- lm(rentals ~ temperature, data = train_set)# ② テストセット（未知データ）で予測predictions <- predict(model_train, newdata = test_set)# ③ 実測値と予測値の相関からR²を計算r_squared <- cor(test_set$rentals, predictions)^2
📌 結果の読み方：テストセットのR²が、訓練セットのR²とだいたい同じなら過学習していません。モデルがノイズに惑わされず、シグナルを捉えている証拠です。完璧なモデルなら、実測値と予測値が一直線に並び R²＝1 になります。R²は cor() を二乗するか、専用パッケージでも計算できます。
🍎 かんたん説明

set.seedは「サイコロの出目を固定する魔法」。これがないと、実行するたびに違う分け方になってしまいます。固定すれば、誰がやっても同じ結果＝再現できるので、チームで安心して結果を共有できます🎲

⑤

データ漏洩（データリーケージ）に注意

分割の「前」と「後」、どっちでデータを掃除する？

とても重要なテーマです。「データのクリーニングは、分割の前にやる？後にやる？」

分割の「前」にやってOKなこと

  • 列名を直す
  • 重複行を削除する
  • 明らかなデータ入力ミスを直す

分割の「後」にやるべきこと（テストの情報を使うもの）

多くの処理は分割の後、訓練セットだけで決めて、その結果をテストセットにも同じように適用します。

⚠️ データ漏洩（data leakage）とは：テストセットの情報が訓練セットに漏れてしまうこと。これが起きると、性能が実際より良く見えてしまう（過度に楽観的）のです。

例：欠損値を全データの中央値で埋めると、テストセットのデータが中央値に影響し、それが訓練セット→モデルに漏れます。これがデータ漏洩です。NAの補完（imputation）は分割の後に、訓練セットだけで決めましょう。

訓練セットとテストセットの間にはしっかりした仕切り（パーティション）を保つこと。モデルは「訓練データだけ」で学習し、「テストデータ（未知データ）」で評価する——この境界を守るのが鉄則です。

データ漏洩は「模擬試験の問題をこっそり見ながら勉強する」ようなもの。それで模擬試験が満点でも、本番では意味がないですよね。テストセット（模擬試験）は絶対に勉強に混ぜない——これがデータ漏洩を防ぐコツです🚧
