🩺 診断（Diagnostics）

モデルは作れた。でも「これは良いモデル？」を、どう判断する？1つの指標に頼らず、調整済みR²・AIC・BIC・S・RMSE・F統計量・MallowのCpを総合的に見る方法を学ぶ。

①

なぜ「診断」が必要なのか

予測変数4つのモデルより、2つのモデルのほうが良いこともある

重回帰では、たくさんの予測変数の候補があります。でも「4つ入れたモデル」より「2つだけのモデル」のほうが良いことも珍しくありません。では、どの予測変数を使うべきか、本当に良い予測ができているのかを、どう判断するのでしょう？

⚠️ p値だけに頼らない！ 統計でよくある失敗は、summaryの最後にある星（★ ★★ ★★★）＝小さなp値だけを見て「有意だからOK」と決めてしまうこと。診断は1つの指標に頼らず、複数を総合して（ホリスティックに）判断するのが鉄則です。

この章では、モデルAとモデルBを比べる代表的な診断指標を学びます。例として2つのモデルを比較します。

モデルA（単回帰）
気温だけ
vs
モデルB（重回帰）
気温＋湿度＋風速
→
どっちが良い？
指標で判定
🍎 かんたん説明

診断は、お医者さんの「健康診断」みたいなもの。体温だけ見て健康とは言えませんよね。身長・体重・血圧・心拍…いろんな数値を合わせて見るから、本当の健康状態がわかります。モデルも同じで、いろんな指標を合わせて「良いモデルか」を診断します🩺

②

調整済みR²（高いほど良い ⬆️）

無駄な変数を入れると「ペナルティ」がかかる賢いR²

ふつうのR²には弱点があります。変数を増やすだけで、無意味な変数でもR²は上がってしまうのです。たとえば家の価格を予測するのに「ドアノブの数」や「ドアノブの平均直径」を入れても、R²は上がります（でも価格とは関係ない！）。

そこで重回帰では調整済みR²を使います。これは役に立たない変数を入れるとペナルティを与えて、水増しされたR²を下げる賢い指標です。

# 調整済みR²を取り出す（高いほうが良い）summary(temp_model)$adj.r.squared # 気温だけsummary(bikes_model_1)$adj.r.squared # 気温+湿度+風速
気温だけ : 0.50（くらい） 気温+湿度+風速 : 0.52（くらい）← 増えた！

重回帰モデルのほうが調整済みR²が高くなりました。つまり湿度と風速を知ることで、レンタル数の予測がよくなるということです。

R² と 調整済みR² の違い（無駄な変数を足すと…）
R²は変数を足すほど必ず上がる。調整済みR²は「役に立つ変数」では上がり、「無駄な変数」では下がる。
🍎 かんたん説明

R²は「荷物をいくら詰めても文句を言わないカバン」。調整済みR²は「いらない荷物を入れると重くなって怒るカバン」。だから本当に必要なものだけ入れるようになります🎒

③

AIC と BIC（低いほど良い ⬇️）

「当てはまりの良さ」と「シンプルさ」のバランス

AIC（赤池情報量規準）

AIC（Akaike Information Criterion）は、「データへの当てはまりの良さ」と「モデルの複雑さ」のバランスを取ろうとする指標です。データによく合うけど、複雑すぎて過学習（オーバーフィット）にならないモデルを好みます。

  • 覚えること①：当てはまりと複雑さのバランスを取る。
  • 覚えること②：AICは低いほうが良い ⬇️。
  • AICは相対的な指標。数値の絶対値（例：10,000）には意味がなく、他のモデルと比べるためだけに使う。
AIC(temp_model) # 気温だけAIC(bikes_model_1) # 気温+湿度+風速 ← こちらが少し低い＝良い

BIC（ベイズ情報量規準）

BIC（Bayesian Information Criterion）はAICの仲間ですが、複雑さへのペナルティがもっと強いのが特徴。よりシンプルな（パーシモニアスな）モデルを好みます。Rでは大文字で BIC() と書きます（小文字だとエラー）。

BIC(temp_model)BIC(bikes_model_1) # こちらが低い＝重回帰モデルが好まれる
📌 AIC vs BIC：どちらも「低いほど良い」。違いはBICのほうがシンプルさを強く求めること。たくさんの変数を入れたモデルだと、BICはAICより少ない変数を選びがちです。
🍎 かんたん説明

AICとBICは「テストの採点者」。当てはまりが良いと加点、でも答案が長すぎ（複雑すぎ）ると減点します。BICは特に厳しい先生で、「もっと短く、シンプルに！」と求めます。点数（AIC/BIC）は低いほど良い＝ゴルフのスコアと同じ⛳

④

残差標準誤差 S と RMSE（低いほど良い ⬇️）

予測が実際の点からどれだけ離れているか

残差は「回帰線から観測点までの距離」でしたね。この距離が大きいほど誤差が大きい。だから残差標準誤差 S は小さいほど良い ⬇️。すべての点が線の近くにある＝良いモデルです。

# summary の "Residual standard error" がS（例: 1509）summary(temp_model)$sigmasummary(bikes_model_1)$sigma # こちらが小さい＝良い

RMSE（二乗平均平方根誤差）

Rには RMSE の組み込み関数が無いので自作します。やることは名前の通り：残差（実際−予測）を二乗 → 平均 → 平方根（Root-Mean-Square-Error）。最後に平方根を取るのは、二乗で変わった単位を元に戻すためです。

calculate_rmse <- function(actual, predicted) { residuals <- actual - predicted # 残差 sqrt(mean(residuals^2)) # 二乗→平均→平方根}calculate_rmse(bikes$rentals, bikes_model_1$fitted.values)
📌 S と RMSE の関係：大きなデータセットでは、S と RMSE はほぼ同じ値に近づきます（例：1509 と 1507）。どちらも重回帰モデルのほうが小さく、より良いことを示しました。
🍎 かんたん説明

ダーツで的の中心からどれだけ外れたか、を平均したのがRMSE。中心に近いほど（＝Sが小さいほど）上手なプレイヤー＝良いモデルです🎯

⑤

F統計量（モデル全体の検定）

「予測変数は全部ゼロ＝無意味」を否定できるか

summaryの最後に出るF統計量は、モデル全体の当てはまりを検定します（ANOVAと同じ考え方で「オムニバス検定」と呼ばれます）。

  • 帰無仮説 H₀：すべての係数がゼロ＝どの予測変数も応答変数と統計的に有意な関係がない。
  • 対立仮説 H₁：少なくとも1つの係数はゼロではない。
  • Fが大きい＝予測変数を入れたモデルのほうが、入れないモデルよりよく当てはまる。
  • p値が有意水準（α）より小さければ帰無仮説を棄却。
⚠️ 大事な注意：F統計量が有意でも、すべての予測変数が有意とは限りません。これは「モデル全体」を見るオムニバス検定。「少なくとも1つは有意」とわかるだけで、どれが一番効いているかはさらに分析が必要です。
summary(bikes_model_1)$fstatistic# 分子と分母の自由度も一緒に返ってくる
🍎 かんたん説明

F統計量は「このチーム、誰か1人でも仕事してる？」という質問。答えが「YES（少なくとも1人は働いてる）」だとわかるけど、誰が一番のエースかまではわからない。それは個別に調べる必要があります⚽

⑥

MallowのCp（予測変数の数と比べる）

過学習・過小学習を見抜く

MallowのCp（Mallows' Cₚ）は、計算した値を予測変数の数と比べて判断します（数えるときは切片も含めます）。

Cp の値意味
Cp ≒ 予測変数の数✅ 良いモデル。過度に複雑にせずデータをよく説明している。
Cp > 予測変数の数⚠️ 過学習（オーバーフィット）
Cp < 予測変数の数⚠️ 過小学習（アンダーフィット）

Rには組み込み関数が無いので自作します。例では、気温モデルも重回帰モデルも Cp＝予測変数の数（2） となり、どちらも良いモデルと判定されました。

📋 診断指標 まとめ

指標良いのは何を見る
調整済みR²高い ⬆️無駄な変数を罰した説明力
AIC低い ⬇️当てはまり×複雑さのバランス
BIC低い ⬇️同上（シンプルさをより重視）
S / RMSE低い ⬇️予測の平均的なズレ
F統計量大きい・p小 ⬆️モデル全体が有意か
MallowのCp≒予測変数の数過学習/過小学習
📌 結論：これらは代表的な指標の一部です。1つに頼らず複数を総合して、モデルAとBを比較しましょう。今回はすべての指標で「重回帰モデル」が（少しだけ）良いという結論でした。
🍎 かんたん説明

MallowのCpは「ちょうどいい荷物の量メーター」。荷物が多すぎ（過学習）でも少なすぎ（過小学習）でもダメ。ちょうど予測変数の数と同じくらいが、バランスの取れた完璧な旅行カバンです🧳
