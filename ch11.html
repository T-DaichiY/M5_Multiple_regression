①

ステップワイズ選択 — 両方のいいとこ取り

足すことも、削ることもできる「混合」選択

前章までで前進選択（足す）と後退選択（削る）を学びました。では「両方を同時にやったら？」と思った人——するどい！それがステップワイズ（mixed / 混合）選択です。

ステップワイズは、前進のように予測変数を足し、後退のように予測変数を削る——両方の良いとこ取りです。各ステップで「足すべきか、削るべきか」を判断します。

全部入りモデル
AIC 9160 から
→
各ステップで
足す or 削る
AICが最も下がる操作
↺
最終モデル
AICが最小
📌 一度削った変数も、また足せる：ステップワイズの強みは、後で「やっぱり戻したい」変数を再追加できること。前進選択は足す一方、後退選択は削る一方ですが、ステップワイズは行ったり来たりできます。
🍎 かんたん説明

前進＝「足すだけ」、後退＝「削るだけ」、ステップワイズ＝「足したり削ったり、両方できる」。料理で言えば、味見しながら「塩を足す」も「水で薄める」も自由にできるシェフです👨‍🍳

②

Rでステップワイズを実行

direction = "both" を指定するだけ

前進・後退と同じ step() 関数を使い、direction = "both" を指定します。全部入りモデルから始めるのが一般的です。

# 全部入りモデルから両方向（足す＋削る）で探索model_all <- lm(rentals ~ ., data = bikes_encoded)step(model_all, direction = "both")

処理の流れ：全部入り（AIC 9160）→ 全部入りなのでまず削るしかできません。weekdaySaturday を削るとAICが下がる → real_feel も削る → さらに進めると、途中で「＋（足し戻す）」の選択肢も現れます。各ステップでAICを一番下げる操作（足す or 削る）を選び、もう改善しなくなったら停止します。

📌 ＋と−が混在する：後退選択の出力は「−」だけ、前進は「＋」だけでしたが、ステップワイズの出力には「＋」と「−」の両方が現れます。これが「行ったり来たり」している証拠です。
🍎 かんたん説明

魔法の言葉は direction = "both"。"forward"（前進）でも "backward"（後退）でもなく "both"（両方）と言うだけで、足す・削るを自由に行き来してくれます🔀

③

p値で「入る・出る」基準を決める

olsrr の ols_step_both_p

AICではなくp値を基準にすることもできます。olsrr パッケージの ols_step_both_p を使うと、変数が入る基準（p_enter）と出る基準（p_remove）を別々に設定できます。

library(olsrr)# p値ベースのステップワイズ（入る≤0.2、出る>0.1で除外）ols_step_both_p(model_all, p_enter = 0.2, p_remove = 0.1)
パラメータ意味
p_enter = 0.2変数がモデルに入るには、p値が0.2以下である必要がある
p_remove = 0.1p値が0.1より大きい変数は、モデルから除外される

出力には AIC・BIC・R²・調整済みR²・SBIC など複数の指標が表示されます。例では、real_feel を追加 → 後で削除、windspeed² を追加 → 後で windspeed を削除、Monday を追加 → やはり不要と判断して削除…と、足し引きを繰り返して最終モデルにたどり着きます。

⚠️ モデルは自動保存されない：ols_step_both_p は更新後のモデルを自動では保存しません。選ばれた変数で実際にモデルを使いたいなら、その変数を自分で lm() に書いて新しいモデルを作る必要があります。また、表示されるのは標準化回帰係数ではなく、ふつうの係数です。
🍎 かんたん説明

p_enter と p_remove は「クラブの入場・退場ルール」。「p値0.2以下なら入っていいよ（入場）」「p値0.1超えたら出てってね（退場）」という2つのドアの基準を別々に決められます🚪

④

階層的選択 — 専門知識で順番を決める

統計だけでなく「理論」で変数を入れる

階層的回帰（hierarchical regression）は、前進・後退・ステップワイズとは発想が違います。これらが変数を独立に・統計的に扱うのに対し、階層的選択は理論的・概念的な構造を尊重して変数を順番に入れていきます。

ポイントはドメイン知識（その分野の専門知識）。データに詳しい専門家が「この変数を先に入れるべき」と判断し、重要度や理論的関係に基づいて順序を決めます。

例：会社の「仕事のパフォーマンス」を予測する

ステップ1：基本属性を入れる
年齢・学歴
↓
ステップ2：仕事関連の変数を追加
経験・スキル
↓
ステップ3：交互作用項を追加
年齢×経験 など

このように、まず土台となる変数（年齢・学歴）を入れ、次に直接関係する変数（経験・スキル）、最後に交互作用——と意味のある順番で積み上げます。これには、その分野とデータへの深い理解が必要です。

🍎 かんたん説明

前進・後退・ステップワイズは「コンピュータが数字（AIC・p値）だけで自動で選ぶ」やり方。階層的選択は「その道のプロが、経験と理論で『この順番で入れよう』と決める」やり方。家を建てるとき、まず土台→柱→屋根、と順番が決まっているのと同じです🏗️

⑤

4手法まとめ ＆ パーシモニー（簡潔さ）

オッカムの剃刀：シンプルが一番

変数選択 4手法の比較

手法動き判断基準過学習リスク
前進選択足すのみ統計（AIC等）あり
後退選択削るのみ統計（AIC/p値）あり
ステップワイズ足す＋削る統計（AIC/p値）あり
階層的選択理論で順に追加ドメイン知識・理論低め（理論ベース）

パーシモニー（parsimony）＝簡潔さを目指す

変数選択のゴールはパーシモニアスなモデル＝できるだけシンプルかつ効果的にデータを説明し、シグナルを捉えつつ予測変数の数を最小にしたモデルです。

🪒 オッカムの剃刀（Occam's Razor）：「すべての条件が同じなら、よりシンプルな説明（モデル）が好ましい」。たとえば、3変数のモデルと2変数のモデルのAICがほぼ同じなら、2変数のモデルを選ぶ。データの保存・収集が少なくて済み、解釈もしやすく、過学習も避けやすいからです。
  • 効率：変数が少ない＝集めるデータが少なくて済む。
  • 解釈性：少ない変数のほうが人間に理解しやすい。
  • 汎化：変数が少ないほど、未知データへの一般化がうまくいき、過学習を避けられる（過学習は第12章で学ぶ）。
🍎 かんたん説明

「オッカムの剃刀」は「同じ結果なら、シンプルなほうがエライ」というルール。旅行カバンも、同じ荷物なら小さいほうがラクですよね。モデルも、同じ性能なら変数が少ないほうが扱いやすくて賢いのです🎒
