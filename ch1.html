Chapter 1

📈 重回帰分析（Multiple Regression）

1つの手がかりだけで予測するのが「単回帰」。複数の手がかりを同時に使うのが「重回帰」。自転車レンタル数を例に、モデルの作り方と予測の仕方をゼロから学ぶ。

① 単回帰のおさらい ② 重回帰とは ③ Rでモデルを作る ④ 係数と回帰式 ⑤ 予測の3つの方法 📝 Quiz
🍎 むずかしいと感じたら？ 図・アニメ・スライダーで“特性だけ”をつかめる ▶ 第1章 超やさしい版 を先に読むのがおすすめです。
①

まずは単回帰のおさらい

1つの予測変数で予測する一番シンプルな形

このモジュールではこれまで単回帰（単純線形回帰 / 最小二乗回帰）を学びました。式はこんな形でしたね。

# 単回帰の式（1つの予測変数 X だけ）Yᵢ = B₀ + B₁·Xᵢ + (誤差)# 例: レンタル数 = 切片 + 傾き×気温 + 誤差
  • Xᵢ：予測に使う手がかり（予測変数）。例：気温。
  • Yᵢ：予測したい値（応答変数 / 目的変数）。例：その日の自転車レンタル数。
  • B₀：y切片、B₁：傾き。
  • 誤差：推定にはどうしてもズレ（ばらつき）がある。誤差Eの代わりに「Ŷ（ワイハット＝予測値）」で書くこともできる。
🍎 かんたん説明

単回帰は「アイスの売上は気温だけで決まる！」と決めつけて予測するようなもの。たしかに気温は大事だけど…雨の日や曜日も関係するよね？それを全部使おうというのが、次の「重回帰」です。

②

重回帰とは — たくさんの手がかりを使う

気温だけじゃなく、湿度も風速も全部使って予測する

重回帰はこう言います。「気温だけからレンタル数を予測するんじゃなくて、もっとたくさんの予測変数を使おうよ」。式は単回帰によく似ていますが、予測変数が増えます。

# 重回帰の式（予測変数が n 個に増える）Y = B₀ + B₁·X₁ + B₂·X₂ + B₃·X₃ + … + Bₙ·Xₙ + 誤差

最初の部分は単回帰そっくりですが、X₂、X₃…と第n番目の予測変数まで続けられます（添え字は予測変数 Predictor の頭文字Pで書くこともあります）。たくさんの手がかりを使うことで、応答変数Y（＝レンタル数）の推定がよりよくなります。

🍎 かんたん説明

探偵が事件を解くとき、手がかりは1つより多いほうが正しい犯人に近づけますよね。重回帰は「湿度」「気温」「風速」という複数の手がかりを集めて、より正確に「今日のレンタル数」を当てる探偵です🕵️

③

Rで重回帰モデルを作る

じつはとても簡単。+ でつなぐだけ

Rで重回帰モデルを作るのは驚くほど簡単です。応答変数 ~ 予測変数1 + 予測変数2 + … と書くだけ。~（チルダ）は「〜で説明する／〜が与えられたとき」という意味です。

# rentals（レンタル数）を 湿度+気温+風速 で予測するbikes_model_1 <- lm(rentals ~ humidity + temperature + windspeed, data = bikes)bikes_model_1 # 係数を表示

実行すると、係数（coefficients）が出てきます。今回は予測変数が3つなので B₁, B₂, B₃ が得られます。これがあなたの重回帰モデルの式になります。さらに詳しい情報を見たいときは summary() を使います。

summary(bikes_model_1)
Coefficients: Estimate (Intercept) ... ← B₀（切片） humidity -3100 ← B₁ temperature ... ← B₂ windspeed ... ← B₃

「Estimate（推定値）」の列に並ぶのが、回帰式に使う係数です。それ以外の部分は単回帰の summary とほぼ同じ。ただ予測変数の行が増えただけです。

📌 4変数の可視化テク：3つの予測変数＋1つの応答変数＝合計4つの変数は、plotlyライブラリで1つの3D散布図にできます。X・Y・Z軸に湿度・気温・風速、点の色をレンタル数にすると、ドラッグして回転できるインタラクティブな図になります。気温が低いと（青＝レンタル少）、高いと（黄＝レンタル多）という関係が立体的に見えます。
④

係数と回帰式 — 予測の準備

3つの数字を教えてくれれば予測できる

得られた係数を使うと、回帰式はこう書けます。

予測レンタル数 (rental_hat) = 切片(B₀) + B₁ × 湿度(humidity) + B₂ × 気温(temperature) + B₃ × 風速(windspeed)

つまり、ある日の湿度・気温・風速の3つさえ教えてもらえれば、その日のレンタル数を予測できるということです。

⚠️ 係数の大きさ＝重要度ではない！ 「湿度の係数が一番大きいから湿度が一番大事」とは言えません。なぜなら湿度・気温・風速は単位（スケール）が違うからです。公平に重要度を比べる方法は第3章（標準化回帰係数）で学びます。
🍎 かんたん説明

回帰式は「料理のレシピ」です。「湿度を◯倍、気温を◯倍、風速を◯倍して全部足す」というレシピ通りに計算すれば、その日のレンタル数という料理ができあがります🍳

⑤

予測の3つの方法

手計算 → fitted.values → predict（だんだん便利に）

実際にモデルで予測してみましょう。例として15番目の観測（15日目）を使います。その日は寒い日で、実際のレンタル数は1,248件でした。予測の方法は3つあります。

方法1：手計算（一番長いやり方）

回帰式に数字を直接入れて計算します。Rでは係数とカッコの間に必ず *（かけ算）が必要です（数学のように省略できません）。

# 15日目の 湿度・気温・風速 を式に代入pred <- B0 + B1*0.49 + B2*(気温) + B3*(風速)# → 予測 約3,300件
📊 残差（ざんさ）の話：予測は約3,300件、実際は1,248件。差がとても大きいですね。これを残差と呼びます。予測が実際より大きい（過大評価）ので、これは負の残差（実際−予測がマイナス）です。この日についてはモデルがあまり役立っていない、ということがわかります。

方法2：fitted.values（データにある行だけ）

モデルが計算済みの「あてはめ値（予測値）」を取り出します。15番目を見たいなら：

bikes_model_1$fitted.values[15]# → 3,324（15日目の予測値）
⚠️ fitted.values の制限：これはデータフレームに実際に存在する行の予測しかできません。「湿度0.65・気温38・風速7」のような、データに無い組み合わせの予測には使えません。

方法3：predict（一番便利・おすすめ）

predict() は手計算を自動でやってくれます。データに無い好きな値でも予測できます。注意点は、新しいデータはデータフレームで、列名を正しく指定すること。

# データに無い好きな値で予測new_data <- data.frame(humidity = 0.65, temperature = 38, windspeed = 7)predict(bikes_model_1, newdata = new_data)# → 2,058件

複数の予測を一度にしたいときは c()（連結）で値を並べます。湿度が高め・気温が高め・風速が低めの日なら、レンタル数の予測はぐっと大きくなります。

🍎 かんたん説明

3つの予測方法は「電卓のレベルアップ」みたいなもの。手計算は紙と鉛筆、fitted.valuesは「過去のテスト結果一覧表」から探すだけ、predictは「どんな数字を入れても答えを出す万能電卓」。だから普段は predict を使えばOK！

📘 重要用語

重回帰分析
Multiple Regression
2つ以上の予測変数を使って、1つの量的な応答変数を予測する手法。
予測変数
Predictor / Independent variable
予測の手がかりとして使う変数（X）。説明変数・独立変数とも呼ぶ。
応答変数
Response / Dependent variable
予測したい対象（Y）。目的変数・従属変数とも呼ぶ。例：レンタル数。
係数
Coefficient (B₀, B₁…)
各予測変数にかける数字。summaryのEstimate列に出る。
残差
Residual
実際の値 − 予測値。マイナスなら過大評価、プラスなら過小評価。
predict関数
predict()
データに無い値でも予測できる、最も便利な予測方法。
