Chapter 5

〰️ 多項式回帰（Polynomial Regression）

線形回帰は「直線の関係」を前提にする。でも気温とレンタル数は“ある気温まで増えて、暑すぎると減る”曲線。直線では捉えられない関係を扱う方法を学ぶ。

🍎 むずかしいと感じたら？ 図・アニメ・スライダーで“特性だけ”をつかめる ▶ 第5章 超やさしい版 を先に読むのがおすすめです。
①

直線の限界

線形回帰の大前提は「直線の関係」

線形回帰の主な前提は、予測変数と応答変数の関係が直線（線形）であること。でも現実はいつも直線とは限りません。対処法は2つあります。

  • データを変換する：対数変換などで、データを直線パターンに近づける。
  • 非線形の関係を使う（この章）：多項式回帰で、曲線そのものをモデルに取り込む。
🍎 かんたん説明

アイスの売上を考えてみて。寒い→売れない、暖かい→よく売れる、でも暑すぎると外に出たくなくて逆に減る。これは直線じゃなく「山なりの曲線」。直線の定規ではこの山を測れません🍦

②

曲線を見つける（geom_smooth）

散布図に滑らかな線を引いて、形を確かめる

geom_smooth を使い、各予測変数（湿度・風速・気温）を個別にレンタル数と散布図にすると、関係の形が見えます。

ggplot(bikes, aes(x = temperature, y = rentals)) + geom_point() + geom_smooth() # 滑らかな曲線で関係を可視化
気温とレンタル数の関係（直線では足りない！）
気温が約65度まではレンタル増。暑すぎると減る → 放物線（2次曲線）の形。直線では捉えられない。

結果：湿度はやや曲線、風速もわずかに曲線、気温ははっきりした曲線パターンでした。気温は約65度まで上がり、暑すぎると減る——放物線（2次）のような形です。

🍎 かんたん説明

geom_smoothは「点々を結ぶ、なめらかなロープ」。点をバラまいてロープを乗せると、それが直線なのか山なりなのか一目でわかります。気温は見事に「山」の形でした⛰️

③

2乗項（多項式）を加える

予測変数を2乗して、曲線を表現する

放物線のような曲がりを捉えるため、各予測変数を2乗した新しい列を作ってモデルに加えます。これが多項式回帰です。

bikes <- bikes %>% mutate(humidity2 = humidity^2, windspeed2 = windspeed^2, temperature2 = temperature^2)bikes_model_3 <- lm(rentals ~ humidity + temperature + windspeed + humidity2 + temperature2 + windspeed2, data = bikes)
モデル予測変数調整済みR²
気温のみ1個0.50
モデル2気温+湿度+風速0.52
モデル3（多項式）上記＋3つの2乗項増加！（より説明できた）
📌 ただし注意：モデル3では windspeed2（風速の2乗）のp値が大きく、有意ではありません（星なし）。散布図でも風速は一番曲がりが弱かったですね。だから風速の2乗は不要かもしれません。
🍎 かんたん説明

直線しか描けないクレヨンに、「2乗」という魔法をかけると、急にカーブが描けるようになります。気温の山も、湿度のゆるいカーブも、これでバッチリ描けます🖍️

④

後退消去（いらない項を削る）

有意でない予測変数を取り除く

有意でない予測変数を取り除くことを後退消去（backwards elimination）と呼びます（詳しくは第10章）。ここでは有意でなかった windspeed2 を削ってみます。

# windspeed2（風速の2乗）を取り除くbikes_model_3b <- lm(rentals ~ humidity + temperature + windspeed + humidity2 + temperature2, data = bikes)# → 残りの予測変数はすべて有意（星つき）に！

削った結果、すべての予測変数が有意になり、R²もわずかに上がりました。無駄な項を取り除いて、すっきり良いモデルになりました。

🍎 かんたん説明

料理に入れた調味料の中で「これ、味に関係ないな」というもの（windspeed2）を取り除く作業。余計なものを抜くと、かえって味（モデル）が引き締まります🧂

⑤

多項式回帰のVIF問題

気温と気温²は当然相関する

多項式回帰には避けられない問題があります。VIF（許容度）を計算すると、気温と気温²、湿度と湿度²が多重共線性を示します。VIFが非常に大きくなるのです。

⚠️ なぜ起きる？ 気温²は気温から作った変数なので、自動的に相関します（気温なしに気温²は計算できない）。これは構造上当たり前のことです。
📌 重要なポイント：多項式回帰でVIFが高くても、モデルがよく機能し、未知のデータ（テストセット）でもうまく一般化するなら、この共線性を必ずしも除く必要はありません。気温²のような自然な相関は許容されることが多いのです。

気になるなら、湿度や気温の項を削って調整済みR²が改善するか試せます。例では湿度を削っても調整済みR²はほぼ変わらず（改善せず）。一方で湿度と風速は、他の変数が捉えていないシグナルをうまく捉えていました。

🍎 かんたん説明

「身長」と「身長の2乗」が似ているのは当たり前ですよね（同じ人の数字だから）。これは“悪い共線性”ではなく“自然な親子関係”。テストでちゃんと点が取れる（よく機能する）なら、この親子はそのまま残してOKです👨‍👦

📘 重要用語

多項式回帰
Polynomial Regression
2乗などの項を加え、曲線（非線形）の関係を捉える回帰。
非線形
Non-linear
直線でない関係。例：気温が高すぎるとレンタルが減る山なり。
geom_smooth
geom_smooth()
ggplotで滑らかな曲線を描き、関係の形を可視化する。
2乗項
Quadratic term (x²)
予測変数を2乗した項。放物線状の関係を表現できる。
後退消去
Backwards Elimination
有意でない予測変数を取り除く手順（第10章で詳説）。

💡 ポイント

曲線が見えたら x² の項を追加。

有意でない2乗項は削る（後退消去）。

x と x² のVIFが高いのは自然。よく機能するなら無理に消さなくてOK。
