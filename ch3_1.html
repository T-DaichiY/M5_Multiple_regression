Chapter 3.1

🎯 影響点分析（Influential Points）

「外れ値」と「影響点」は別もの。回帰線を大きくゆがめる“悪い影響を与える点”を、クックの距離（Cook's D）で自動的に見つける方法を学ぶ。

🍎 むずかしいと感じたら？ 図・アニメ・スライダーで“特性だけ”をつかめる ▶ 第3.1章 超やさしい版 を先に読むのがおすすめです。
①

外れ値の3つのタイプ

外れ値だからといって、必ず悪さをするとは限らない

大事な前提：すべての外れ値が「影響点」ではありません。外れ値の現れ方には3タイプあります。

外れ値の3タイプ（赤い点＝外れ値）
①②は回帰線をゆがめる可能性大。③は外れ値だが線の傾向に沿うので影響は小さい。
  • ① Y方向の外れ値：Xは平均的だがYが大きくズレている。残差が大きい。
  • ② X方向の外れ値（高レバレッジ点）：Yは平均的だがXが他から大きく離れている。回帰線をグイッと自分のほうへ引っ張る力が強い。
  • ③ 線の傾向に沿う点：XもYも極端だが、回帰線の延長線上にある。除いても線はほぼ変わらない＝影響は小さい。
🍎 かんたん説明

クラスの平均身長を考えるとき、すごく背が高い子（外れ値）がいても、その子が「みんなの傾向」から外れていなければ平均はあまり変わりません。でも、たった1人だけ全然違う場所にいる子は、平均をグイッと動かしてしまう。これが「影響点」です🏫

②

影響点とは何か

回帰線の式や統計量を大きく変えてしまう点

影響点（influential point）とは、回帰線の係数・標準誤差・調整済みR²などの統計量に大きな影響を与える1組の (X, Y) のことです。高レバレッジ点②のように、回帰線を自分のほうへ引っ張ってしまう点が典型です。

影響点かどうかを調べる方法

  • 手動：その点を入れた回帰と、除いた回帰の両方を作り、式がどれだけ変わるか比べる。→ でも候補が100個あったら大変！
  • 自動（推奨）：クックの距離（Cook's distance）を使う。これが次の主役です。
🍎 かんたん説明

影響点を手で探すのは「砂浜から特定の貝殻を1つずつ拾って確かめる」ようなもの。大変すぎます。だから、自動で見つけてくれる「金属探知機」＝クックの距離を使うのです🏖️

③

クックの距離（Cook's D）

点を1つ取り除いたら、どれだけ変わる？を測る

クックの距離は、「ある観測を除いたとき、モデルがどれだけ変わるか」を測ります。点を1つ取り出して全部を再計算し、しきい値と比べるだけ。値が大きいほど、その点はモデルに大きな影響を与えています。

このクラスで使うしきい値

# このクラスの基準: 4 / (N - K - 1) より大きければ影響点しきい値 = 4 / (N - K - 1)# N = 観測数（行の数）# K = 予測変数の数

例：自転車データは N＝731行、予測変数 K＝3 なので、しきい値は 4/(731−3−1) ≈ 0.0055。これより大きいCook's Dの点を影響点とみなします。

別のしきい値（覚えやすい版）判定
D > 0.5影響点の可能性あり
D > 1.0かなり高い確率で影響点
D > 4/N予測変数の数を使わない簡易版
🍎 かんたん説明

クックの距離は「その人がいなくなったら、クラスの雰囲気がどれだけ変わるか」のメーター。1人抜けても何も変わらない人もいれば、その人が抜けると一気に変わる“ムードメーカー”もいる。後者が影響点です🎭

④

Rで影響点を探す

OLSRRパッケージで一目瞭然

OLSRR パッケージにはCook's Dを可視化する関数があります。棒グラフで全データ点を表示し、しきい値の線を引いてくれます。

library(olsrr)ols_plot_cooksd_bar(my_model) # 棒グラフ（赤=影響点, 青=正常）ols_plot_cooksd_chart(my_model) # もう少しきれいな版

例では、69行目がしきい値をはるかに超える巨大なCook's Dを示しました。調べてみると…

69行目（2011年3月10日）: ・祝日ではない ・3月の日 ・レンタル数 = たった623件（とても少ない） ・湿度 = 0（異常！通常の最小は0.18） ・風速 = かなり速い（四分位より上） ・気温 = 低め → モデルの予測は5,000件超。実際は623件 → 巨大な残差！
📌 分析者の視点：湿度0%は明らかに異常です。データ入力ミスの可能性を疑い、「2011年3月10日について別の情報はないか」と会社に確認するのが実務的な対応です。むやみに削除せず、まず原因を考えましょう。

影響点を一覧で取り出す

# Cook's D を大きい順に上位10件head(sort(cooks.distance(my_model), decreasing = TRUE), 10)# しきい値0.0055を超える点のインデックスcooks_d <- cooks.distance(my_model)influential <- unique(which(cooks_d > 0.0055))

この基準で29個の影響点が見つかりました。

⑤

影響点を除いてモデルを比較

本当に良くなった？を確かめる
# 影響点を除いたデータ（先頭の -（マイナス）に注目）bikes_no_infl <- bikes[-influential, ]# どれだけデータが残ったかnrow(bikes_no_infl) / nrow(bikes) # → 0.96（96%が残る = 4%を除外）

702行が残り、96%のデータを保持（4%を影響点として除外）。どれだけデータが失われるか把握するのは大切です。除去前後でモデルを比べると…

項目全データ（除去前）影響点を除去後
係数の推定値元の値変化した（影響があった証拠）
調整済みR²0.45より高い（改善！）
📌 結論：影響点を除いたら調整済みR²が上がりました。これはモデルがシグナルをよりよく捉え、ノイズを無視できるようになったことを意味します。Cook's Dの判断通り、除去は有益でした。
🍎 かんたん説明

クラス写真を撮るとき、1人だけ変な動きでブレている子がいると全体がぼやけて見える。その子（影響点）を外して撮り直すと、みんながくっきり写る＝モデルがクリアになる、というイメージです📸 ただし「外す前になぜブレたのか」を確認するのを忘れずに！

📘 重要用語

影響点
Influential Point
回帰線の係数や統計量を大きく変えてしまう (X,Y) の点。
外れ値
Outlier
他の点から大きく離れた点。必ずしも影響点とは限らない。
高レバレッジ点
High Leverage Point
X方向に遠い外れ値。回帰線を引っ張る力が強い。
クックの距離
Cook's Distance (Cook's D)
点を除いたときのモデル変化量。影響度の自動判定に使う。
OLSRR
olsrr package
Cook's Dの可視化など、回帰診断に便利なRパッケージ。

💡 ポイント

外れ値 ≠ 影響点。

Cook's D が 4/(N−K−1) を超えたら影響点の候補。

削除する前に「データ入力ミスでは？」と原因を確認する習慣を！
