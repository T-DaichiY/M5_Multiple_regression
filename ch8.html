✖️ 交互作用（Interaction）

運動だけ、食事だけでもコレステロールは下がる。でも両方を組み合わせると、効果は単なる足し算以上になる。2変数の「組み合わせ効果」をモデルに入れる方法を学ぶ。

①

交互作用とは — 組み合わせ効果

1+1が2より大きくなることがある

ときどき、2つの変数が「組み合わさったとき」に応答変数へ特別な効果を生むことがあります。これが交互作用です。身近な例：

目標単独でも効く組み合わせると…
コレステロールを下げる運動 / 食事運動＋食事でさらに大きな効果
血圧を改善する薬 / 生活習慣両方の組み合わせでより大きな改善
生産性を上げる研修 / 経験研修＋経験の両方が欲しい

つまり、個々でも良いけれど、2つの交互作用は単独の効果を足し合わせた以上になることがあるのです。

🍎 かんたん説明

カレーのルウだけ、ごはんだけでもおいしい。でも「カレーライス」にすると、おいしさが単なる足し算を超えますよね。これが交互作用＝組み合わせの魔法です🍛

②

多重共線性との違い

似ているようで、まったく別もの
多重共線性（第4章）交互作用（この章）
2変数の関係互いに相関している（似ている）必ずしも相関していない
問題か効果か⚠️ 問題（係数が不安定に）✅ 効果（予測がよくなる）
ポイント変数同士が被っている変数を組み合わせた効果が応答に効く

交互作用では、2つの変数は互いに関連していなくてもよいのです。組み合わせたときに、別々にしておくのとは違う効果が応答変数に現れる——これが交互作用です。

🍎 かんたん説明

多重共線性は「同じことを言う双子」（被ってて困る）。交互作用は「性格の違う2人がチームを組むと、すごい力を発揮する」（組み合わせが強い）。似て非なるものです👥

③

Rでの書き方（* と :）

風速×天気の組み合わせ効果を入れる

自転車の例では、天気が悪化し、かつ風速も上がったら、それぞれ単独より大きくレンタル数に効くと考えられます。Rで交互作用を入れるには * を使います。出力では交互作用が :（コロン）で表示されます。

# 交互作用なしのモデルmodel <- lm(rentals ~ windspeed + weather, data = bikes_clean)# 交互作用あり: * は両方を個別＋交互作用で含める近道model_int <- lm(rentals ~ windspeed * weather, data = bikes_clean)summary(model_int)
📌 * と : の違い：windspeed * weather は「windspeed」「weather」「windspeed:weather（交互作用）」の3つすべてを自動で含める近道。出力では交互作用項が windspeed:weatherHeavy のようにコロンで表示されます。
🍎 かんたん説明

*（アスタリスク）は「2人を別々に呼んで、さらにペアとしても呼ぶ」という便利な記号。:（コロン）は出力の中で「これがペアの効果だよ」と教えてくれる印です✖️

④

交互作用つきの式で予測する

交互作用項にも値を代入する

交互作用つきモデルの式はこうなります（係数は例）。天気は「大雨か？」「小雨か？」のYes/No（0/1）で、両方0なら晴れの日です。

予測 = 切片 - 132 × 風速 - 1878 × 大雨(0 or 1) - ... × 小雨(0 or 1) + 係数 × (風速 × 大雨) # ← 交互作用項 + 係数 × (風速 × 小雨)

例：風速10mph・晴れの日（大雨も小雨もNo）

風速＝10、大雨＝0、小雨＝0。すると交互作用項（風速×大雨など）はすべて0になります。計算すると約4,579台のレンタル予測。

💡 実務のヒント：手計算は理解のためで、実際は predict() 関数や自作関数を使うほうが効率的です。
🍎 かんたん説明

交互作用つきの式は「材料費＋ペアセット割引」みたいなもの。風速と天気を別々に足すだけでなく、「風速×天気のペア効果」も計算に入れる。だから現実に近い予測ができます🧮

⑤

交互作用の効果を確かめる

風と雨が「重なる」と予測が大きく変わる

風速（軽い1mph vs 強い10mph）と天気（晴れ・小雨・大雨）を組み合わせて予測を比べると、交互作用の威力がわかります。

条件予測レンタル数
強風(10) ＋ 晴れ4,579
強風(10) ＋ 小雨3,815 付近
強風(10) ＋ 大雨1,940（大きく激減！）
弱風(1) ＋ 大雨3,815（強風時よりずっと多い）
交互作用：風と雨が重なると予測が激減
弱風だと天気が悪くても緩やかに減る。強風だと天気の悪化で急降下。2本の線の傾きが違う＝交互作用がある証拠。

ポイント：風だけ・雨だけで見ると予測はそれぞれ違いますが、2つの交互作用を考えると、別々に見たときとは大きく異なる予測が得られます。線の傾きが違うことが、交互作用の存在を示しています。

📌 補足：交互作用は線プロットでも判定できます（2本の線が平行でなければ交互作用あり）。またANOVAも交互作用を扱います。
🍎 かんたん説明

「弱い風＋大雨」より「強い風＋大雨」のほうが、自転車を借りる人がずっと減りますよね。風と雨が重なるとダブルパンチ。これが交互作用。別々に足し算しただけでは、このダブルパンチを見逃してしまいます🌧️💨

📘 重要用語

交互作用
Interaction
2変数を組み合わせたときに生じる、別々とは異なる応答への効果。
* （アスタリスク）
interaction operator
A*B で A・B・A:B の3つを自動で含める近道。
: （コロン）
colon
出力で交互作用項を表す記号（例 windspeed:weatherHeavy）。
線プロット
Line plot
2本の線が平行でなければ交互作用ありと判定できる。

💡 ポイント

交互作用 ＝ 組み合わせ効果（1+1が2以上に）。

共線性とは別もの（こちらは“効果”）。

R： A * B で個別＋ペアを一括。

線が平行でない＝交互作用あり。
